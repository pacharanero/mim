COCT_RM000210UK01
PartBirthTime

Parent class containing and linking to information about an entity which is a part of another entity. In this case, it is the birth time part of a person entity.

If used in an update-to-PDS message (e.g. General Update Request):
In order to control the update semantics, in addition to the attributes below, the following will also be present:
  • An updateMode attribute to indicate whether the item should be "added" or "altered" (mandatory). This item cannot be removed via messaging.
If used in a notification message:
In addition to the attributes below, in order to specify the type of update that has occurred, the following will also be present:
  • An updateMode attribute to indicate whether the item has been "added", "altered", or is "unchanged" (mandatory)

[1..1] [M] birthTime (TS)

To provide the date the patient was born.

The date/time may be given in one of the following formats:

  • YYYYMMDDHHMM (Date and Time)
  • YYYYMMDD (Date Only)

One of the Date and Time or Date Only datatype flavours is used to carry this information as follows:

  • The value attribute will contain the date value in the format indicated.
The format must be as follows, depending on which message this CMET appears in:
  • In an allocate message (Allocate or Primary Care Allocate), only Date Only is permitted
  • In an update (General or Primary Care) or Demographics Notification message, both Date Only and Date and Time are permitted
  • In all other messages, only Date and Time is permitted.
