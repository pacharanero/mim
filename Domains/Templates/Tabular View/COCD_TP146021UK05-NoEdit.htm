The mood code of the service.

  • The code attribute should be set according the context of the information being conveyed and will be selected from the vocabulary.
[1..1] [M] code (CD {CNE:ServicesSnCT } )

A SnomedCT code taken from the subset shown.

The Concept Descriptor (CD) with CDA URL datatype/flavour shall be used to carry this information as follows:

  • The code attribute shall contain a code from the relevant vocabulary
  • The codeSystem attribute shall contain the OID for SnomedCT
  • The displayName attribute shall carry a description associated with the code in SnomedCT
  • The originalText element can be used to carry a reference element. This can link the coded item to the relevant text in the CDA section
  • qualifier (s) may be used if required
Note the implementation guidance for the domain where this template is used should be consulted regarding correct usage of the originalText element.

[1..1] [M] effectiveTime (IVL<TS>)

The low attribute (mandatory) should be used to state when a service has begun or is planned to begin.
The high attribute (optional) should be used to state when the service finished or is planned to finish.

The centre attribute should not be used.

The date range shall be carried as follows:

If both a start date and an end date are present the Date or Time Interval Complete datatype shall be used:

  • The low attribute shall contain the start date
  • The high attribute shall contain the end date.

If only a start date is present the Date or Time Interval After datatype shall be used:

  • The low attribute shall contain the start date.
