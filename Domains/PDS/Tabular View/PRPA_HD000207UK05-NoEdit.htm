PRPA_RM000207UK05
PdsTraceMatch
Parent class linking to details returned in the Advanced Trace Query Response.

As well as confirming that the NHS number exists on the PDS, a number of data fields are also retrieved from the PDS. Historic data may optionally also be searched, and/or returned. Current and future dated data is always returned. Multiple matching records may be returned, along with a MatchingLevel (this will only be populated for an algorithmic search) for each match, indicating the confidence of the match.

  • The code attribute will contain a value from the list of codes in the InformationSensitivity vocabulary;

[0..1] multipleBirthOrderNumber (INT)
Describes the sequence in the case of multiple births (e.g. twins, triplets etc).

For births that are not a part of a multiple birth, this will be set to 1.

Note that permissible values are 1-9, with any value between

  • 1 and 7 inclusive indicating the actual birth order number,
  • 8 meaning "Not applicable" and
  • 9 meaning "Not known".
