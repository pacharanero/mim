Constraint Type Status Detail
Goal 1. Goal Approved
Create a document about an individual patient/ service user, which is suitable to be shared with another professional.
Description 2. Description Approved
The user creates a document on their local application, which relates to an individual patient/ service user.
Trigger 3. Trigger Approved
User has carried out (or is currently carrying out) a consultation/ assessment with a person, which is documented on the local application. (Documentation Begins).
Pre-condition - Subject of Document 4. Pre-condition Approved
Document relates to a single person.
Pre-condition - NHS Number 4. Pre-condition Approved
User has traced the person on PDS and has obtained the NHS number.
Post-condition - Document Created in Draft 5.Post-condition Approved
The process can end the document being created and left in "draft status", to be completed later.
Post-condition - Document Created and Finalised 5.Post-condition Approved
The process can end with the document being created and finalised - no further changes to the document are required.
Post-condition - Document Not Created (User choice) 5.Post-condition Approved
After looking at existing information / documents for this person, User decides not create a document.
Post-condition - Document Created and made available for Authorisation 5.Post-condition Approved
The process can end with the document being created and made available to another user to authorise the content.
Post-condition - LR Failure 5.Post-condition Approved
If an LR does not exist and the creation of a self-claim LR fails the activity will stop.
Post-condition - RBAC Failure 5.Post-condition Approved
If the user does not have the required RBAC then activity will end with a failure.
