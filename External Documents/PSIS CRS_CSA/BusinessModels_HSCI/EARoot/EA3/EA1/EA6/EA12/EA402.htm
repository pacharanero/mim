The CarePlan template may be classified to map local terminology to MIM standards, to allow a receiving system to re-use the information within a care plan.

For example:
A local system may use the term "My Personal Preferences" or "Expressed Preferences" to describe any particular wishes a person has stated about their care and services. Both of these terms could be mapped to the standard code "PRE" - "Care Preferences".

The system should use the vocab TextHeadingType to indicate which part of a care plan each element of text refers to.

The code can be used in the CODE attribute on the HL7 CDA TextSection template to identify the type of text section.

The code can also be used in the XML ID attribute, to form part of a unique identifier of a text element or any of its children (e.g. paragraph, table cell etc). Note IDs must be unique within the document, therefore the convention "Code_n" should be adopted.
For example:
PRE_1, PRE_2 etc.
