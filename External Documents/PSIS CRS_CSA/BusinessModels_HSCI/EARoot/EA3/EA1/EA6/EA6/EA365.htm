If an assessment document contains a single score on a specified assessment scale, the system must record the score in the VALUE attribute on the Assessment template, the scale should be recorded in CODE attribute on the Scale act.

If an assessment document contains multiple scores, the system must record the score in the VALUE attribute on the AssessmentComponent act, the scale should be recorded in CODE attribute on the Scale act.

Worked example: (a) single score on a scale, (b) multiple scores

(a) Waterlow pressure sore assessment (Summary only)
Assessment template (focal act):
- code attribute 225393005 | pressure sore assessment
- value attribute = 14
Scale:
- code attribute = 278897004 | Waterlow pressure sore risk score.

(b) Waterlow pressure sore assessment (Summary and Detail)
Assessment template (focal act):
- code attribute 225393005 | pressure sore assessment
- value attribute = 14
Scale:
- code attribute = 278897004 | Waterlow pressure sore risk score

AssessmentComponent:
- code attribute = 416686006 | continence assessment
- value attribute = 0
Scale:
- code attribute = 206021000000103 | Waterlow continence score

AssessmentComponent:
- code attribute = 225397006| skin assessment
- value attribute = 2
Scale:
- code attribute = 206081000000102 | Waterlow skin type visual risk areas score

(and so on - a new component for each part of the assessment)

This applies to:
- Contact Assessment document
- Overview Assessment document
- Specialist assessment Outcome document
