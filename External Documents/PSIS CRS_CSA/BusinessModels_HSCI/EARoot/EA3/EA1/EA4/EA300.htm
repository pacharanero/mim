Suppliers should consider and consult with their user groups regarding the system requirements for the different users that required access to Health and Social Care Integration documents.

They need to consider which users will need their system to provide them with the ability to:
1) Create and send HSCI documents
2) Receive HSCI documents

For example:

It may be appropriate for a GP system to be able to CREATE the Contact Assessment Document, and RECEIVE all other document types. It may be appropriate for a social care management system to CREATE and RECEIVE all document types, etc.
