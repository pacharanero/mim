1. Introduction



1.1 About This Document

This document is a report which has been generated from the Requirements Catalogue element of the Health and Social Care Integration Analysis Model (the Model). The Model version number, status and File CM reference can be found on the cover page of this document. The Model contains requirements and design specifications expressed in narrative form and UML (Unified Modelling Language) notation.
The Model can be viewed on the Health and Social Care Integration SharePoint site or the HTML version can be downloaded from the external instance of File CM.


 

1.2 Audience

This document should be read by all suppliers involved in the implementation of the Health and Social Care Integration (HSCI) messages contained within Message Implementation Manual (MIM) 7

The requirements contained within this document may also be of relevance to the National Integration Centre (NIC) and care communities (Local Authorities, NHS Trusts etc) implementing the HSCI domain messages.


 

1.3 Background

NHS Connecting for Health (NHS CFH) is an executive agency of the Department of Health and is responsible for delivering the National Programme for Information Technology (NPfIT).

The NPfIT is a programme aimed at the long term, government led vision of a unified NHS IT system. This will be made possible through a highly developed "Information and Communication Technology" (ICT) platform that will enable patient record details to be accessed anywhere in England using the patient's unique NHS number.

The solutions developed by NHS CFH will be implemented incrementally, giving patients greater choice, more involvement in decisions about their own care, and access to their own records over time.

The Health and Social Care Integration Domain addresses the requirement for joined up services across organisations through the use of joint and shared assessments and care plans.



 

1.4 Purpose

This document provides an understanding of use and creation of the Health and Social Care Integration domain 2008-B CDA Documents.

This Requirements Catalogue should not be read in isolation - please read in conjunction with the Business Analysis Model:
NHS CFH ID NPFIT-SHR-MODL-HSC-0011

And

NPfIT Message Implementation Manual (MIM) 7

These requirements should inform suppliers in designing their solutions, and may be used by those creating acceptance and testing criteria for systems implementing the HSCI domain.


 

1.5 Definitions

The keywords MUST, MAY, and SHOULD are to be interpreted as described in RFC2119:

MUST: This word, or the terms "REQUIRED" or "SHALL", means that the definition is an absolute requirement of the specification.

SHOULD: This word, or the adjective "RECOMMENDED", means that there may exist valid reasons in particular circumstances to ignore a particular item, but the full implications must be understood and carefully weighed before choosing a different course.

MAY: This word, or the adjective "OPTIONAL", mean that an item is truly optional.  One implementer may choose to include the item because a particular implementation requires it or because the implementer feels that it enhances the implementation while another implementer may omit the same item.  An implementation which does not include a particular option MUST be prepared to interoperate with another implementation which does include the option, though perhaps with reduced functionality.  In the same vein an implementation which does include a particular option MUST be prepared to interoperate with another implementation which does not include the option (except, of course, for the feature the option provides.

The system suppliers wishing to comply with the Health and Social Care Integration requirements will be referred to as "the system" throughout this document.

In some cases there may be requirements which are specific only to social care systems - and do not apply to NHS systems. In this case the term "the social care management system" will be used.

- Contact Assessment document
