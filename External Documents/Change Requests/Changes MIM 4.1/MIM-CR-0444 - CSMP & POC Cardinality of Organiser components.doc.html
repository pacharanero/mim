Short Title of Issue

Change of cardinality on component ActRelationship in CSMP and POC.

 

Description

As part of the example messaging project the need to identify a consistent representation of common clinical observations has been identified. For use where there may be one or more components and the number is unpredictable. The cardinality of the component ActRelationship in the CSMP is currently 2..*. In order to model group of test as an Organizer, with individual tests and results as components, this cardinality requires changing to 0..*. It is likely that other test groups will have a similar requirement.

 

Perceived Impact

Minor – This will have little or no effect on existing message designs that claim conformance to the CSMP.

 

Reason for issue being raised

In order to be able to Group (and other combined) tests consistently as an Organizer.

 

Suggested Action

Change the cardinality of the component ActRelationship in the CSMP to 0..*.

 

A constraint should also be placed on the component ActRelationship in the CSMP stating that this relationship should only be used where it is applicable to model an Organizer or one of its valid refinements.
