Short Title of Issue

Some PSIS Query example messages reference the wrong schema version.

 

 

Description

The PSIS CRE Query (QUPC_MT100101UK06), PSIS Clinical Statement Query (QUPC_MT110101UK05) and PSIS DPA Subject Access Query (QUPC_MT120101UK05) example messages reference the wrong schema versions (QUPC_MT100101UK05.xsd, QUPC_MT110101UK03.xsd and QUPC_MT120101UK03.xsd respectively). It appears that this has been the case in all MIM versions from 3.1.08 onwards.

 

 

Perceived Impact

Correcting the example messages to reference the correct schema version should not negatively impact any domain as they are only referenced by the PSIS Query domain. Not correcting them back will cause the problem to persist.

 

Backward Compatibility Status

 

Reason for issue being raised

Bugzilla issue #13986 raised by BT NASP.

ALSO #14094 and 14092

 

Suggested Action

Correct the example messages to reference the correct schema versions (QUPC_MT100101UK06.xsd, QUPC_MT110101UK05.xsd and QUPC_MT120101UK05.xsd respectively) and check they still validate against the new version schemas.
